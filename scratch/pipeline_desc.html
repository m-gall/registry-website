About the standard

{{ subheading1 }}

Components of the pipeline descriptions:

  • A structured document with a pipeline description which borrows CWL terminology, fields and descriptors
  • A complementary markdown document containing additional unstructured text descriptions
  • A graphical representation of the pipeline rendered using CWL explorer from the CWL document

    • Structured document elements:

    • Objects: Packages, versions, data & file inputs/outputs, databases, genome versions, parameters
    • Descriptions: Definitions and purpose with links to other standards (e.g. EDAM format) where applicable
    • Usage: Use cases for documents, evaluation checkpoints, pipeline exits, report provenance

      • Visual representation elements:

      • Packages and connections between inputs/outputs
      • Workflow and expandable subworkflows

        • Document storage:

        • Each centre has a Github repository containing markdown and CWL documents
